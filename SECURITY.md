# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest (main) | Yes |
| older releases | No |

Only the latest release on the `main` branch receives security fixes. Please update to the latest version before reporting an issue.

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Report vulnerabilities by email to **security@assettrack.com**. Please include:

- A clear description of the vulnerability
- Steps to reproduce or a proof-of-concept
- The potential impact and affected versions
- Any suggested mitigations (optional)

You will receive an acknowledgement within **48 hours**. A patch or mitigation will be delivered within **7 days** for confirmed issues. We will coordinate a disclosure date with you and ask that you refrain from public disclosure until the fix has been released.

## Scope

In-scope: authentication and authorisation logic, data exposure, injection vulnerabilities, dependency issues with known CVEs.

Out-of-scope: denial-of-service attacks requiring physical access, issues in third-party services outside our control.

## Disclosure Policy

We follow a coordinated (responsible) disclosure model. Please give us reasonable time to address the issue before any public disclosure. We will credit reporters in release notes unless anonymity is requested.
